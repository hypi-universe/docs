Hypi GraphQL CRUD Tutorial

In this tutorial, let's check how to use GraphQL APIs to perform CRUD operations on your app.

For the purpose of this tutorial, we‚Äôve created an app with the name hypi-tutorials, and in the GraphQL Editor we‚Äôve created a simple schema for our application. Check here how to add schema using GraphQL Editor.

interface Employee{
name: String!
age: Int!
colleagues: [Employee!]
}
type Actor implements Employee{
name: String!
age: Int!
colleagues: [Employee!]
starsIn: [Episode!]
}
type Director implements Employee{
name: String!
age: Int!
colleagues: [Employee!]
hasDirected: [Episode!]
}
enum Episode{
S01E01
S01E02
S01E03
S01E04
S01E05
S01E06
}

Now, with a created instance of the app, navigate to the GraphQL Playground of the instance, or do it programmatically with a http request to:

https://api.hypi.app/graphql

with the http headers

"Authorization": "", - your authorization token
"hypi-domain": "" ‚Äìthe domain of your Hypi app

Let‚Äôs start performing some operations on your Hypi app. For the purpose of this tutorial, we will use the GraphQL Playground

Please check Documentation for the GraphQL queries and mutations mentioned in this tutorial.

Create#

For this tutorial, we will create an Actor object with the name ‚ÄúWill Smith‚Äù, age 17, and he stars in episodes 1, 2 and 3. To create an object we will use upsert method. The mutation will look like this.

mutation {
upsert(values: {
Actor: {
name: "Will Smith",
age: 17,
starsIn: [S01E01, S01E02, S01E03]
}
}
) {
id
}
}

As you can see, if you are familiar with GraphQL , the call is pretty simple, you pass in the object that you want to create as a values parameter to the upsert mutation, and specify what fields of the Magic Hypi object you want in the response.

You can look into all of the fields that the Magic Hypi object contains under its section in the Hypi GraphQL API documentation, here we just return the id.

As expected, the response looks like this

{
"data": {
"upsert": [
{
"id": "01ED4X75AG1G96PDC8A231KTC8"
}
]
}
}

So, we have successfully used the upsert method to create an Actor in our database.

Next, let‚Äôs see how we can update that actor.

Update#

To update an existing value, we should provide the hypi.idfor the object along with the data that we want to save for that object.

Let‚Äôs say that we want to update the age of the actor we just created.

The mutation will look like this.

mutation {
upsert(values: {
Actor: {
age: 18,
hypi: {
id: "01ED4X75AG1G96PDC8A231KTC8"
}
}
}
) {
id
}
}

As you can see, the difference between the create and the update is that the update contains

hypi: {
id: "01ED4X75AG1G96PDC8A231KTC8"
}

Here, hypi object id is the one that we got back in the create response . As expected, the response remains the same as Create.

Read#

For reading the data, we will show you two queries (get and find) that Hypi provides out of the box for any object.

  1. get()

The get query provides a way to get a single object by its hypi.id. The query will be like this.

query {
get(type: Actor, id: "01ED4X75AG1G96PDC8A231KTC8") {
... on Actor {
name
age
}
}
}

As you can see the get query accepts type and id as parameters and it returns whatever fields you specify for the type that you are using it for.

The response from this query will be

{
"data": {
"get": {
"name": "Will Smith",
"age": 18
}
}
}

As expected, we got the name and the age of the Actor with id ‚Äú01ED4X75AG1G96PDC8A231KTC8‚Äù.

  1. find()

Thefind query is used to search through your data. For the purpose of this tutorial, we‚Äôve created a few more actors in the database.

The find query accepts two mandatory parameters Type and arcql. The Type is the type of object you want to query, and the arcql parameter is the query that you want to use for searching though the data.

To learn more about Arcql , please refer ArcQl filtering.

In this tutorial, we will cover listing of all of the actors and searching them by name.

For getting all of the actors we will send the wildcard ‚Äú*‚Äù as the arcql parameter, indicating that we want all of the actors. In the response, we want the name of the actor along with his hypi.id. The query will look like this.

query {
find(type: Actor, arcql: "*") {
... on HypiFilterConnection {
edges {
node {
... on Actor {
name
hypi {
id
}
}
}
}
}
}
}

As expected, the response is a list of all of the edges of type Actor

{
"data": {
"find": {
"edges": [
{
"node": {
"name": "Will Smith",
"hypi": {
"id": "01ED4X75AG1G96PDC8A231KTC8"
}
}
},
{
"node": {
"name": "Alfonso Ribeiro",
"hypi": {
"id": "01ED4ZCASYQ57YK2VV3WCM3D0E"
}
}
},
{
"node": {
"name": "Will Werrell",
"hypi": {
"id": "01ED506E9GQ3Z75RHE0609RNYN"
}
}
}
]
}
}
}

Next, we will use Arcql to create a typical auto-complete query for the name field. The arcql will look like this.

‚Äúname ^ ‚ÄòWill‚Äô‚Äù

And the query will look like this.

query {
find(type: Actor, arcql: "name ^ 'Will'") {
... on HypiFilterConnection {
edges {
node {
... on Actor {
name
hypi {
id
}
}
}
}
}
}
}

As expected, the response contains only the actors with the names starting with ‚ÄòWill‚Äô

{
"data": {
"find": {
"edges": [
{
"node": {
"name": "Will Smith",
"hypi": {
"id": "01ED4X75AG1G96PDC8A231KTC8"
}
}
},
{
"node": {
"name": "Will Werrell",
"hypi": {
"id": "01ED506E9GQ3Z75RHE0609RNYN"
}
}
}
]
}
}
}

Delete#

Hypi offers soft delete and hard delete of objects.

  1. Soft Delete

Soft deleting an object in the Hypi Universe , is referred to as ‚Äòtrashing‚Äô an object.

The field ‚Äútrashed‚Äù, which is a part of the Magic Hypi object is used to track that.

The trash mutation accepts two parameters, the Type of the object and an arcql query. It will trash up to 25 of the objects that match the query. The mutation call looks like this.

mutation {
trash(type: Actor, arcql: "hypi.id='01ED4X75AG1G96PDC8A231KTC8'")
}

The trash mutation returns the number of trashed items

{
"data": {
"trash": 1
}
}

We can verify that the item has been trashed by calling the get query with the id of the trashed object.

It should not return any data.

You can find include the trashed items in searches by calling find with the parameter‚ÄúincludeTrashed: True‚Äù

The untrash mutation accepts two parameters, the Type of the object and an arcql query. It will untrash all of the objects that match the query. The 'get' query with the 'id' would return the data as before.

The mutation call looks like this.

mutation {
untrash(type: Actor, arcql: "hypi.id='01ED4X75AG1G96PDC8A231KTC8'")
}

The untrash mutation returns the number of untrashed items

{
"data": {
"untrash": 1
}
}
  1. Hard Delete

Hard delete is done by calling the delete mutation. The delete mutation accepts two parameters, theTypeof the object and an arcql query. It will delete up to 25 of the objects that match the query. The mutation call looks like this.

mutation {
delete(type: Actor, arcql: "hypi.id='01ED4X75AG1G96PDC8A231KTC8'")
}

The delete mutation returns the number of deleted items

{
"data": {
"delete": 1
}
}

Once an object has been hard deleted, there is no way to recover it.
